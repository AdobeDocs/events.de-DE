# Einfrieren, Wiederherstellen, Innovieren: Praktisch mit Snapshots der schnellen Entwicklungsumgebung für die Cloud-Entwicklung

Erfahren Sie, wie schnelle Entwicklungsumgebungen (RDEs) in Adobe Experience Manager Cloud Service schnelle, Cloud-native Iterationen ermöglichen. Erfahren Sie, wie Entwicklerinnen und Entwickler mit Snapshot-Funktionen vollständige RDE-Status einfrieren, wiederherstellen und wiederverwenden können. Dadurch können Sie mit dem Adobe I/O CLI RDE-Plug-in Tests optimieren, das Rollback sofort durchführen und Entwicklungszyklen beschleunigen. Verbessern Sie die Effizienz Ihrer Cloud-Entwicklung und stimmen Sie sich an den Richtlinien zur Unternehmensführung ab.

>[!VIDEO](https://video.tv.adobe.com/v/3477705/?learn=on&enablevpops)

Diese Sitzung — Freeze, Restore, Innovate: Hands-On with Rapid Development Environment Snapshots for Cloud Development — beinhaltet Remo Liechti, Software Development Engineer bei Adobe, und wurde für Adobe Developers Live 2025 voraufgezeichnet. Remo zeigt, wie schnelle Entwicklungsumgebungen (RDEs) eine schnelle, Cloud-native Iteration in Adobe Experience Manager Cloud Service ermöglichen und wie neue Snapshot-Funktionen es Entwickelnden ermöglichen, vollständige RDE-Status einzufrieren, wiederherzustellen und wiederzuverwenden. Erfahren Sie, wie Sie mit dem Adobe I/O CLI RDE-Plug-in Tests optimieren, das Rollback sofort durchführen und Entwicklungszyklen in AEM Cloud beschleunigen können.
